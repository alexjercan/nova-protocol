## Goal

Third playtest round (2026-07-12): overshooting a beacon is hard to
recover from, so [X] STOP should be taught in beat 1; objective changes
are easy to miss (add a sound for complete/new, plus a brief visual);
the scavenger should keep the fight at the debris field instead of
roaming. Verdicts verbatim in Notes.

## Steps

- [x] (finding 1) Teach [X] in beat 1: b1 objective text becomes
      "...hold [W] to burn, tap [X] to stop." (and drop the [X] line
      from b3, which taught it late). Text-only; the walk test's
      objective-content asserts do not key on these strings.
- [x] (finding 2a) Objective change sounds: two new NovaSfx keys
      (ObjectiveNew, ObjectiveComplete) + placeholder WAVs generated by
      extending scripts/gen-placeholder-sounds.py (short single blip for
      new; rising two-tone for complete), wired through NOVA_SFX_FILES
      and the SoundBank. A nova hud system diffs GameObjectives ids
      against a Local snapshot each frame (the resource is write-on-diff
      since 20260711-180506 R1.1, so resource_changed gating is cheap
      and real): removed id -> complete cue, added id -> new cue.
      Non-positional one-shots (UI sounds, no attenuation).
- [x] (finding 2b) Objective completed VFX: when an id disappears from
      GameObjectives, spawn a transient "ghost" line - the completed
      message in a distinct done-green, fading out over ~2.5s, then
      despawned. NOT a child of the bcs panel (rebuild_lines
      replace_children would despawn it mid-fade) - an absolute node
      stacked beside/below the panel. Test: complete an objective, ghost
      exists with the message; advance past the fade, ghost gone; the
      new-objective line itself is untouched.
- [x] (finding 3) AI leash: `AIControllerConfig` gains
      `leash: Option<f32>`; ships with a patrol route get an `AILeash {
      center, radius }` (center = patrol centroid). VERIFY FIRST how the

      is beyond the leash radius from center, break off (return to the
      passive directive: patrol home); re-engage is naturally allowed
      once back inside AI_ENGAGE_RANGE of a hostile. The shakedown
      pirate gets leash Some(~150u) around the debris cluster. Test at
      the AI-decision level: an engaged leashed ship beyond the radius
      falls back to patrol; an unleashed one keeps engaging (delivery
      guard).
- [x] Docs (CHANGELOG; scenario-system.md AI config note) + full
      new-test set + check with --tests --examples + fmt.
