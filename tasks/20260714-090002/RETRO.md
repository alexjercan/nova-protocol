# Retro: Salvage crate pickup polish (20260714-090002)

Closed 2026-07-15. Verdict APPROVE after one MAJOR found and fixed in-cycle.

## What shipped

- `NovaSfx::SalvagePickup` cue: a light rising "ding" (sweep 640->860 Hz,
  0.10s) generated by the existing Python placeholder script, played at 0.22
  volume (under the 0.30/0.38 objective chimes).
- The cue fires from nova_scenario's `SalvageCratePlugin` (which owns
  `SalvageCrateMarker`; nova_gameplay's audio module cannot see it), gated to
  the player and deduped per crate.
- Shakedown crates re-spaced ~29-37u -> >=53u apart, pinned by a test.

## What went well

- Reading the whole audio module + the collision/area pipeline before touching
  anything meant the design (where the observer lives, why it can't be in
  nova_gameplay, non-positional vs positional) fell out cleanly.
- Self-review caught the real bug instead of shipping it (see below).

## What went wrong / the bug

The first cut observed raw `CollisionStart` gated to the player. That reads as
"one pickup, one ding" until you remember a ship is one `RigidBody` with MANY
section colliders, and avian fires `CollisionStart` **per collider-pair**, not
per body-pair. An empirical probe (3 section colliders through one sensor)
produced 3 dings.

How it was diagnosed: rather than theorize about despawn timing, I checked the
avian 0.7 source (event carries `collider1/2` AND `body1/2`) and then wrote a
throwaway probe test that panicked with `DING_COUNT=3`. That turned a "maybe"
into a fact in one run.

Fix: dedup per crate entity via a `DingedCrates` set (`insert` returns true only
on first contact), pruned by an `On<Remove, SalvageCrateMarker>` observer. This
decouples the cue's correctness from the scenario despawn timing that the
existing `add_one` crate counting quietly leans on.

## Lessons for next time

- **avian `CollisionStart` is per collider-pair, not per body-pair.** Any cue or
  counter keyed off a multi-collider body (a ship = many section colliders) will
  fire N times unless it dedups on the body/entity. The existing scenario crate
  counting has the same latent exposure; it survives only because the pickup
  despawn usually lands before the trailing sections enter. New code should not
  rely on that - dedup explicitly.
- **A 5-line probe test beats a paragraph of reasoning.** `panic!("N={}", ...)`
  against real physics settled the collider-pair question immediately.
