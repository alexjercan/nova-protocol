# Controller-based RCS burn loop sound (player + autopilot ORBIT/STOP)

- STATUS: CLOSED
- PRIORITY: 5
- TAGS: v0.7.0, feature, audio, flight

## Goal

A looping sound that plays whenever the RCS primitive is burning - driven by the
player's SHIFT fine-adjust OR by the autopilot (ORBIT station-keep, STOP/GOTO
terminal settle). It is CONTROLLER-based: authored on the controller section
(the ship's computer, which already owns the lock/radar/safety cues), gated on
the same condition as `rcs_burn_system` (non-zero `RcsIntent` on the ship root +
the controller granting `FlightVerb::Rcs`), so it fires for both drivers. Models
the existing thruster engine-hum loop exactly (one loop entity per distinct
authored handle, volume tracks burn effort, distance-attenuated + smoothed).

## Notes

Model: the thruster engine-hum loop in audio.rs (ThrusterLoopSfx /
ThrusterHumVolume / ensure+compute+apply). Controller sounds pattern:
ControllerSectionSounds + SectionMeshes controller_*_sound fields. RCS burn gate:
flight.rs rcs_burn_system (RcsIntent != 0 + controller grants Rcs). Asset
convention: placeholder generated by gen-placeholder-sounds.py, declared in
base.bundle.ron, referenced self://sounds/. Content is generated + parity-tested
(edit builders, run `content -- gen`, never the .ron directly).

## Close-out (2026-07-18)

Delivered. The RCS loop is authored on the controller section
(`rcs_loop_sound`), snapshotted into `ControllerSectionSounds::rcs_loop`, and
played by a new `ensure/compute/apply_rcs_loop_volume` chain in audio.rs that
mirrors the thruster engine-hum loop but reads the ship-root `RcsIntent` through
each live controller's authored handle, gated on `FlightVerb::Rcs` (the same
check as `rcs_burn_system`). Because both the player modal and the autopilot
write the same `RcsIntent`, the sound fires for SHIFT nudges and for ORBIT/STOP
maneuvers alike. Placeholder asset `assets/base/sounds/rcs_loop.wav` generated
by a new seamless-looping `hiss` synth; declared in the base bundle; base
controllers author it via `SectionMeshRefs`.

Verification: `audio::` suite 30 passed (incl. 3 new: rcs_volume pure fn, loop
plays-then-mutes, silent-without-verb), `content_ron_parity` 2 passed (the
generated `base.content.ron` carries the new field on all four controllers).

Design: chose controller-authored over a global default sound to stay
consistent with the codebase's authored-or-silent model (mods can reship it);
a distinct hiss synth rather than reusing the thruster hum so RCS reads apart
from the main drive by ear. Volume tracks `RcsIntent.length()` up to
`RCS_MAX_VOLUME = 0.22` (under the engine's 0.3 - RCS is a gentle push).
