# v0.3.0 - Where and when it hurts

This release teaches the scenario system to react to lifecycle and place, and
gives combat its first area weapon. Events now fire on state entry and exit and
when a ship crosses into a region of space, the torpedo bay lands as the first
area-of-effect weapon, and a reworked per-section health system gives all that
damage a consistent thing to bite into. Directional and thruster shaders got
sharper on the way out.

## The torpedo bay and blast damage

The **torpedo bay section** is the game's first **area-of-effect weapon**. A
torpedo does not need a pixel-perfect hit: it detonates and deals **blast
damage** to everything inside its radius, so turret fire stays precise and
pointed while torpedoes are about zoning and catching clustered or fragile
targets. It is a section like any other, so it bolts onto a build with the same
physics and mounting rules as a thruster or a turret.

## A health system built for sections

Blast damage only means something with a consistent target, so health is now a
proper **per-section health system**: every section of a ship carries its own
health and both damage flavours feed the same path, a turret round hitting one
section and a torpedo blast hitting every section within its radius. Damage
stays local, so a torpedo that goes off on one side can shred those sections
while the far side survives - where a hit lands matters, not just whether it
landed.

## Events that know where and when

Scenario events gained lifecycle and location awareness. **OnEnter and OnExit
events** fire when a scenario state begins and ends, so a mission can run setup
on entry and teardown on exit without hand-wiring it into game code. A
**zone-entry trigger** fires an event when a ship moves into a defined region of
space, unlocking objectives like "reach the nav point" or "hold this area" as
data through the same filters-and-actions pipeline - new scenario shapes without
new engine code.

## Sharper shaders

The **directional shader** was reworked to make a ship's forward direction far
more legible at a glance, which matters a lot when reading orientation in the
middle of a fight. The **thruster shaders** were extended to support more complex
shapes and animations, so exhaust plumes can be more expressive than the single
cone they started as.
