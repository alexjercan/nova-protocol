# v0.4.1 - Greasing the release rails

A build and CI plumbing patch: nothing changes in the game itself, but the
release pipeline and the checks that gate it get quieter and more reliable. If
you play Nova Protocol this one is invisible; if you build or contribute to it,
read on.

## Release build

The `release-flow` now installs the `x86_64-apple-darwin` std for the pinned
nightly, which fixes the macOS universal build that was missing that target.

## CI

CI consolidated to a single `--features debug` feature set shared across clippy,
tests and examples, so the whole run leans on one Bevy build instead of several.
The cache is now saved even when a job fails, and the windowed examples smoke
runs as a separate, non-blocking step.

Full, terse detail for every one of these lives in
[CHANGELOG.md](https://github.com/alexjercan/nova-protocol/blob/master/CHANGELOG.md).
