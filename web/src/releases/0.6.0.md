# v0.6.0 - Mods over the wire

The release where modding stops being a local-files affair and becomes a
first-class, over-the-wire experience: a static mod portal ships alongside the
game, and you can browse, install, update and uninstall mods from inside the
game on both native and web. Scenarios got a proper picker, the web build got
its particles back on WebGPU, and mod dependencies now resolve on their own.

## The mod portal, end to end

This is the headline. A static **mod portal** now publishes on every deploy -
validated, sha256-hashed bundles served under `/mods/<id>/<version>/` with a
generated `catalog.json`, the first published mod being Gauntlet Run. The
in-game **Explore online** tab makes that portal live: browse it, then install,
update or uninstall mods on native and web, with per-file progress, an offline
catalog fallback, and updates that preserve whether a mod was enabled.

Underneath sits a real install pipeline: a `mods://` asset source and a RON
installed index back a local cache, downloads are staged and verified against
size and sha256, and a bundle is committed only once every file checks out -
so a half-finished download never corrupts your install.

**Mod dependencies now resolve end to end.** Installing a mod auto-pulls any
missing dependencies, enabling one auto-enables its transitive deps, disabling a
mod that something still depends on is refused, and merge order follows
dependency-respecting topological order. It is ids-only for now - no version
constraints yet.

The Mods menu itself is a two-pane, Factorio-style screen: `Installed` and
`Explore online` tabs over a scrollable list with per-row enable checkboxes, and
a details panel rendered from each mod's bundle metadata. Dev and tooling mods
marked `hidden` stay installed and code-enableable but drop out of the
player-facing list.

## A home for scenarios

The main menu gained a **Scenarios picker**: a two-pane overlay listing every
playable scenario - base and mod-added alike - with a details pane and a Play
button. Scenarios can now carry an optional `thumbnail` and a `hidden` flag. A
new `SetSkybox` action lets a scenario swap the skybox cubemap mid-run, deferred
until the new image loads so a bad path just leaves the sky unchanged.

## Particles return to the web

Moving the web build from WebGL2 to **WebGPU** brought the particle effects back
online in the browser - muzzle flash, projectile trails, and the torpedo launch
and detonation bursts now render on the web as they do on native. Browsers
without WebGPU get a clear "WebGPU required" message instead of a black canvas,
plus a heads-up under the landing page's "Play in browser" button.

## Under the hood

Modding event dispatch is now indexed by event name (upstreamed to
bevy-common-systems), measuring 17-24% faster under bursts of 500-5000 handlers
and neutral at the realistic one-event-per-frame rate - with a criterion
benchmark added to keep that honest. Two sibling optimizations were measured and
deliberately deferred as documented insurance. Full, terse detail for every one
of these lives in
[CHANGELOG.md](https://github.com/alexjercan/nova-protocol/blob/master/CHANGELOG.md).
