# v0.2.0 - A reason to fly

The release where Nova Protocol grows past "build a ship and shoot rocks" and
gains a reason to exist: a data-driven game-events and queue system, the first
scenario and modding capabilities built on top of it, and procedurally
generated asteroids that break apart under fire. It is a small update, but it
lays the backbone the whole game still runs on. The
[devlog](../../blog/devlog-2-objectives-enemy-ai-and-asteroids/) tells the fuller
story, including the first enemy AI and the objectives work.

## Game events and a queue system

The headline is a small, data-driven engine: game **events** fired from the
world, filters that decide when a handler should react, and actions that read or
write scenario state - all dispatched through a queue. Because it is loadable
from an asset rather than hardcoded, a win condition becomes data you can author
instead of Rust you have to compile.

## Scenarios and modding

On top of the event engine sits the first **scenario** system, and with it the
first taste of modding: a scenario is just a bag of event handlers, so custom
objectives can be loaded from assets. This is the direct ancestor of the
scenario engine the game runs on today.

## Asteroids with procedural mesh and dynamic destruction

The old sphere asteroids are gone. Asteroids now build from a noise-displaced
**octahedron sphere** so their triangles stay evenly sized, and they support
dynamic destruction so they break apart rather than just vanish when hit.
