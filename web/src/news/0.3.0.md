# v0.3.0 - Where and when it hurts

v0.2.0 gave the game objectives and a data-driven system to describe them.
v0.3.0 makes that system say more interesting things and gives the player a
weapon that is not just a turret. The event system grows lifecycle and location
awareness, the torpedo bay lands as the first area-of-effect weapon, and a
reworked per-section health system gives all that damage a consistent thing to
bite into. Directional and thruster shaders got sharper on the way out.

## The torpedo bay and blast damage

Until now the only way to hurt something was the turret - a hitscan round that
damages exactly what it touches. The **torpedo bay section** is the game's first
**area-of-effect weapon**. A torpedo does not need a pixel-perfect hit: it
detonates and deals **blast damage** to everything inside its radius, which
changes the tactical texture completely. Turret fire stays precise and pointed
while torpedoes are about zoning and catching clustered or fragile targets.
Because it is a section like any other, it bolts onto a build with the same
physics and mounting rules as a thruster or a turret.

## A health system built for sections

Blast damage only means something with a consistent target, so health is now a
proper **per-section health system**: every section of a ship carries its own
health, and both damage flavours feed the same path - a turret round hitting one
section, a torpedo blast hitting every section within its radius. Damage stays
local, so a torpedo that goes off on one side can shred those sections while the
far side survives. Where a hit lands matters, not just whether it landed.

## Events that know where and when

The v0.2.0 modding backbone could react to things happening, but not to
lifecycle or to place. v0.3.0 adds both. **OnEnter and OnExit events** fire when
a scenario state begins and ends, so a mission can run setup on entry and
teardown on exit without hand-wiring it into game code. A **zone-entry trigger**
fires an event when a ship moves into a defined region of space, unlocking
objectives like "reach the nav point" or "hold this area" as data through the
same filters-and-actions pipeline - new scenario shapes without new engine code.

## Sharper shaders

The **directional shader** was reworked to make a ship's forward direction far
more legible at a glance, which matters a lot when reading orientation in the
middle of a fight. The **thruster shaders** were extended to support more complex
shapes and animations, so exhaust plumes can be more expressive than the single
cone they started as.

## Point releases

**v0.3.1** is a maintenance release that pulls the whole stack up to **Bevy
0.19** and gives the project a proper documentation home. The vendored
`bevy_common_systems` copy is gone, now pulled in as a proper git dependency
instead of living in-tree, and alongside it lands an `AGENTS.md` and a new
`docs/` folder covering architecture, the scenario system, sections,
development, and migration notes - the first real home for how the project fits
together. On the render side, a **post-processing camera component** lands as the
seed of a proper post-processing pass.
