# v0.6.0 - Mods over the wire

For five releases, modding Nova Protocol meant dropping files next to the game
and restarting. v0.6.0 is where that changes: a static **mod portal** ships
alongside the game on every deploy, and a new **Explore online** tab lets you
browse, install, update and uninstall mods from inside the game - on native and
in the browser - with dependencies that resolve themselves. Around that
headline, scenarios got a proper picker, the web build got its particles back on
WebGPU, and event dispatch got measurably faster. This is the biggest release so
far; here is the whole of it.

## The mod portal

Every deploy now publishes a **static mod portal** next to the game: each mod is
a validated bundle served under `/mods/<id>/<version>/`, hashed with sha256, and
listed in a generated `catalog.json`. It is just static files - no server, no
database - so it rides the same GitHub Pages deploy the game does. The first mod
on it is **Gauntlet Run**, the wave-survival scenario that used to be baked into
the game.

## Explore online: mods from inside the game

The **Explore online** tab makes that portal live. Browse the catalog in-game,
then install, update or uninstall any mod without leaving Nova Protocol - the
same on native and on the web. Installs show **per-file progress**, an **offline
catalog fallback** keeps the list usable when the portal is unreachable, and
updates **preserve whether a mod was enabled** so refreshing a mod never quietly
turns it off. Together with the local-install work below, this completes the
mod-portal arc: discovery, install, update and removal all happen over the wire.

## A real install pipeline

Underneath Explore online sits an install pipeline built to not corrupt your
game. Downloaded mods live in a local cache backed by a new `mods://` asset
source and a RON **installed index**, and they load and merge through the exact
same pipeline as the mods that ship with the game - a downloaded mod is not a
second-class citizen. Installs are **staged and verified** against both size and
sha256, and a bundle is **committed only once every file checks out**, so a
half-finished or tampered download never becomes a half-installed mod.

## Dependencies that resolve themselves

Mods can now depend on other mods, and v0.6.0 resolves those dependencies **end
to end**. Installing a mod auto-pulls anything it is missing; enabling a mod
auto-enables its transitive dependencies; disabling a mod that something else
still depends on is refused rather than silently breaking the dependent; and
merge order follows **dependency-respecting topological order** so a mod always
merges after the mods it builds on. It is ids-only for now - version constraints
are still to come.

## The Mods menu, rebuilt

The Mods menu is now a two-pane, **Factorio-style** screen: `Installed` and
`Explore online` tabs over a scrollable list with a per-row enable checkbox, and
a details panel on the side rendered from the selected mod's bundle metadata.
Dev and tooling mods can mark themselves `hidden` - they stay installed and
enableable by id from code, but drop out of the player-facing list so the menu
stays about mods players actually chose.

## A home for scenarios

The main menu gained a **Scenarios picker**: a two-pane overlay listing every
playable scenario - base and mod-added alike - with a details pane and a Play
button, so mod-added scenarios are discoverable instead of hidden behind a
console. Scenarios can now carry an optional `thumbnail` and a `hidden` flag. A
new **`SetSkybox`** action lets a scenario swap the skybox cubemap mid-run,
deferred until the new image finishes loading so a bad path just leaves the sky
unchanged instead of flashing or crashing.

## Particles return to the web

Moving the web build from WebGL2 to **WebGPU** brought the particle effects back
online in the browser - muzzle flash, projectile trails, and the torpedo launch
and detonation bursts now render on the web exactly as they do on native.
Browsers without WebGPU get a clear **"WebGPU required"** message instead of a
black canvas, plus a heads-up under the landing page's "Play in browser" button
so nobody stares at a blank screen wondering what broke.

## Faster event dispatch, measured first

Modding event dispatch is now **indexed by event name** (upstreamed to
bevy-common-systems), which measures **17-24% faster** under bursts of 500-5000
handlers and neutral at the realistic one-event-per-frame rate. Crucially, that
number is backed by a new criterion benchmark rather than a hunch - and two
sibling optimizations (filter-key interning, condition-eval compilation) were
measured, found to be noise at realistic event rates, and **deliberately
deferred** as documented insurance rather than shipped on faith.

## Smaller things

The Screenshot Reel capture set no longer ships in the game assets - its
scenario moved into the example that films it, so players and the web build stop
downloading a capture tool. Two fixes round the release out: the Scenarios
picker no longer crashes the renderer on a non-2D thumbnail (those are skipped
with a warning, and images mount only once loaded), and local mod-portal web
testing no longer needs a cross-origin `?portal=` override now that
`scripts/preview-web.sh` serves the portal same-origin, matching production.
