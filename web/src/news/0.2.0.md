# v0.2.0 - A reason to fly

This is the release where Nova Protocol grows past "build a ship and shoot
rocks" and gains a reason to exist. v0.2.0 ships a data-driven game-events and
queue system, the first scenario and modding capabilities built on top of it,
and procedurally generated asteroids that break apart under fire. It is a small
update, but it lays the backbone the whole game still runs on.

## Game events and a queue system

The headline is a small, data-driven engine: game **events** fired from the
world, filters that decide when a handler should react, and actions that read or
write scenario state - all dispatched through a queue. Because it is loadable
from an asset rather than hardcoded, a win condition becomes data you can author
instead of Rust you have to compile. "When an asteroid is destroyed, increment a
counter; when the counter hits its target, switch scene" is an entire objective
expressed as data.

## Scenarios and modding

On top of the event engine sits the first **scenario** system, and with it the
first taste of modding: a scenario is just a bag of event handlers, so custom
objectives can be loaded from assets. A HUD lists the active objectives and
<kbd>Enter</kbd> advances to the next scenario when one is queued. This is the
direct ancestor of the scenario engine the game runs on today.

ship. Those are devlog color; the shipped features of v0.2.0 are the event
engine, scenarios, and the asteroids below.

## Asteroids with procedural mesh and dynamic destruction

The old sphere asteroids are gone. Asteroids now build from a noise-displaced
**octahedron sphere** so their triangles stay evenly sized rather than bunching
at the poles, and they support dynamic destruction so they break apart rather
than just vanish when hit.

## Point releases

**v0.2.1** is aimed squarely at modders: the modding capabilities from v0.2.0
finally get **documentation and worked examples** to match, so hooking into the
game no longer means reading the source. The **event system** underneath also
got a cleanup refactor - the same events and queue, reworked to be easier to
reason about, with no player-facing change.
