# Craft ships as reusable prototypes (base + mods + menus)

Follow-up to [craft-ships-into-base](craft-ships-into-base.md). That task built
the racer/cargob ships as INLINE section configs in the base scenario builders.
This task turns them into reusable base SECTION PROTOTYPES so that BOTH the base
campaign AND downloaded mods can build these ships by referencing prototype ids -
and re-skins the menus + the gauntlet/the-ledger webmods onto them.

## Why prototypes

Mods (`webmods/gauntlet`, `webmods/the-ledger`) are hand-authored RON with no code
generation. Inlining an 18-section racer / 43-section cargob into every mod ship
would add thousands of unreadable lines. As base prototypes, a mod ship is a
compact list of `(id, position, rotation, source: Prototype("racer_cube_..."))` -
and it references the base's meshes/sounds through the prototype, so mod files
carry NO mesh paths at all.

## What changed

- **`crates/nova_assets/src/scenario/craft.rs`** now exposes:
  - `racer_prototypes(meshes)` / `cargob_prototypes(meshes)` -> `Vec<SectionConfig>`:
    one player-grade prototype per cut cube (`racer_cube_*`, `cargob_cube_*`,
    `cargob_core_controller`), plus a weak `racer_light_cube_*` turret variant per
    turret cube for AI enemies.
  - `racer_sections(grade, controller_mods)` / `cargob_sections()` now return
    `Prototype` references instead of inline configs. Enemy grade swaps the
    turrets to `racer_light_*` and `SetHealth`-nerfs hull/thruster/controller;
    player grade uses the full prototypes as-is.
- **`sections.rs::build_sections`** appends the racer + cargob prototypes to the
  base catalog, so they ship in `base.content.ron` and resolve for base content
  AND any mod that depends on base.
- **Base campaign** is unchanged in behavior: the shakedown/broadside builders
  still call `racer_sections`/`cargob_sections`, which now emit prototype refs.
  `content gen` regenerated the scenario `.ron`.
- **Menu backdrops** (`menu_ambience`, `menu_waystation`, `menu_scrapyard`) now
  fly the racer (`craft::racer_sections`) instead of the old trainer cube - a
  detailed silhouette drifting the backdrop. The `backdrop_orbiter` helper's
  `extra_hull` knob is now a no-op (every orbiter is a full racer).
- **Webmods re-skinned** to the racer/cargob art (prototype refs), by role:
  - Player ships (Kestrel, gauntlet player) -> player-grade racer; their `turret`
    input key is rebound to the two racer turret cubes.
  - Light corvette enemies (magpie_1/2, channel_magpie_1/2) -> weak enemy racer.
  - The-ledger ch2b heavies (Crowbar/Kettle Black) -> FULL-power racer (the
    post-checkpoint difficulty step over the wave-one mooks).
  - Neutral scout/haulers (magpie_scout, dray_mule) -> player-grade racer.
  - The final boss `auditor` -> cargob (the only cargob; user chose "boss only").

## Tests

Base structural tests that keyed on the old inline kinds / prototype ids were
updated to resolve `Prototype` refs against the base catalog
(`scenario_generation::build_section_catalog`) and read the section KIND, or to
read grade from the prototype-id prefix (`racer_light_` = weak scavenger turret).
Updated: `shakedown` ship-shape + goto-withheld, `scenario` menu-orbiter,
`broadside_assault`, `gauntlet_course`, `ledger_ch2_encounter`. Full
`cargo test -p nova_assets` is green; `content lint` clean (base + webmods).

## Notes

- `web/dist/mods/**` is a gitignored build artifact (regenerated by
  `scripts/preview-web.sh`), so the published mod copies need no manual edit.
- Balance numbers (enemy HP/turret, heavy-vs-mook grade split) are first-pass;
  tune in `craft.rs`.
