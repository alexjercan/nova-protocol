# v0.4.0 plan: sections polish + testability

- DATE: 2026-07-07
- BRANCH: feature/cleanup-v0.3.0 (v0.3.1 wrapping up)
- THEME: make the existing ship sections feel good, and make the whole game
  testable, before adding any new gameplay systems.

## Why this scope

The v0.4.0 tag had accumulated ~40 tasks spanning bugfixes, a torpedo overhaul,
an objectives/scenario-language system, modding, HUD work, and a full docs pass.
That is more than one release. v0.4.0 is deliberately narrowed to two things the
current build actually needs:

1. **Polish + bugfix the sections we already ship** - the PDC turret feels clunky
   and the torpedo bay is flaky (self-detonates on spawn, weird controls, weak
   homing).
2. **Make it testable** - dedicated playable test-range examples (with target
   gates, in the style of `~/personal/bevy-common-systems`), reuse of that repo's
   autopilot/screenshot harness so examples double as headless smoke tests, and
   unit/integration tests over the graph + destruction pipeline.

As a byproduct we **catalog which systems are promotion-eligible** for
bevy-common-systems (copy-paste into other games), without doing the cross-repo
moves yet.

Everything feature-shaped (objectives, scenario config language, modding skybox,
torpedo HUD/ammo/visuals, variable damage by type, the big docs pass) has been
retagged `v0.5.0` and is out of scope here.

## In scope (v0.4.0)

Ordered roughly by priority. IDs are tatr task IDs under `tasks/`.

### A. Section bugfixes

- **20260707-100003** (p95, bug) Torpedo self-detonates on spawn when the target
  is near - add an arming gate (min time / min distance) before detonation is
  allowed. This is the headline "torpedo spawns too close and just dies" bug.
- **20260707-100004** (p80, bug) Torpedo despawns silently when its target dies
  mid-flight - degrade gracefully (freeze on last known position) instead of
  vanishing + warn-spamming.
- **20260706-162912** (p85, bug) `OnDestroyed` event fires inconsistently in the
  integrity/destruction path.
