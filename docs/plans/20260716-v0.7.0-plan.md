# v0.7.0 plan: content & polish

- DATE: 2026-07-16
- BASE: master at v0.6.0 (48f0060d)
- THEME: make it a game you can win or lose, with more places to fly. Ship the
  capital-combat vertical slice with an explicit win/lose frame into the base
  game's progression, a story-campaign mod that dogfoods the v0.6.0 authoring
  stack, a real Gauntlet, visual asset variety with mod-shippable resources, a
  measured performance pass, and a set of small, bounded UI/UX improvements
  (settings content, fonts, ammo readout).

## Why this scope

v0.6.0 built the authoring platform (RON scenario format, content model, mod
portal, scenarios picker) but the content written in it is thin: two real
playable scenarios, one portal mod, one asteroid texture, one hull model, and
still no way to win or lose. The user's goals for v0.7.0 are (1) more content
and interesting scenarios/assets, (2) bug fixes and performance, (3) small
UI/UX improvements. Content is the headline: filling the platform with real
scenarios is also the best test of it. Performance follows the house
measure-first rule - v0.6.0 benchmarked the modding layer, but no frame-time
baseline exists for actual gameplay scenes, so the perf strand starts by
producing numbers, not fixes. There are zero open bug tasks, so "bug fixes"
is a release policy (bugs found while building/playtesting content are filed
at p90+ with a `bug` tag and fixed within the release) plus the carried-over
SFX wiring test.

Direction spike: `tasks/20260716-122954/SPIKE.md` (survey of backlog, game,
docs and web state; options weighed: content-first vs authoring-completion vs
platform/input vs maintenance).

## Definition of done

- The game has an explicit win/lose: the capital-combat slice plays start to
  finish with victory and defeat states, and losing your ship means losing.
  It is reachable both through the New Game progression and as its own entry
  in the Scenarios picker.
- The story campaign ships as a downloadable portal mod: multiple chained
  scenarios telling one story, installable from the Explore-online tab; and
  Gauntlet Run 2.0 is re-published as a real obstacle course.
- Scenarios stop all looking the same: new skybox(es), texture variants and a
  model variant ship, mods can carry and reference their OWN resource files
  (textures/models/audio), and every picker entry has a real thumbnail.
- A frame-time baseline report exists for the heavy scenes (native + web);
  every performance change landed in this release cites its numbers.
- The Settings panel is no longer a stub: audio volume, graphics preset and a
  keybind reference work in-game, from both the main menu and pause menu.
- The HUD shows ammo/magazine state; in-game text uses the project's real
  fonts, native and wasm.
- Bugs found during the cycle are fixed at release priority, not queued.
