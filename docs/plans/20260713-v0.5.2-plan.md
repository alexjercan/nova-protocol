# v0.5.2 plan: playtest bugfixes + example testability

- DATE: 2026-07-13
- BASE: master at v0.5.1 (ef1d7ad)
- THEME: fix the two bugs the last playtests surfaced, then convert the
  example suite into a real regression net (rework + CI gate + upstream
  cleanup) and ship a small patch release.

## Why this scope

v0.5.1 shipped the web-build render fixes. What is left on the table from the
playtest logs is small but real: an audio attenuation bug and a teardown
lifecycle warn. The rest of this release is deliberately about testability -
the examples grew organically and the CI smoke gate is currently
non-blocking; v0.5.2 turns that infrastructure into something that catches
regressions instead of tolerating them. Feature work (keybind hint icons,
alt-fire modes) is retagged v0.6.0 and out of scope; sensor-model lock ranges
(20260710-195953) was closed wontdo in the same user pass that set this
scope.

## In scope, in execution order

IDs are tatr task IDs under `tasks/`. Order is bugs first (user's call),
then the rework that pins them, then the gate over the reworked set, then
the upstream cleanup, then the release chore. Priorities encode the same
order (teardown bumped 60 -> 85, rework 90 -> 80 in this pass; substance
unchanged).

1. **20260711-183417** (p90, bug, audio) Thruster hum audible from far away.
   Mechanism verified in source: `update_thruster_loop_volume`
   (crates/nova_gameplay/src/audio.rs:546) drives one global loop from the
   throttle averaged over EVERY thruster in the world, no distance term -
   the code comment defers per-ship attribution, and v0.5.0's AI ships made
   that deferral audible. Fix: per-ship contribution
   `engine_volume * distance_attenuation(listener -> ship)`, max-combined.
   Fail-first App test.
2. **20260712-115902** (p85, bug, flight) `remove::<ManeuverTelemetry>`
   races scenario teardown. The observer's `get_entity` guard checks
   liveness at observer time, not command-apply time; queue
   `try_remove` (verified present in bevy_ecs 0.19) and sweep the race
   class. Fail-first test with the error-handler-to-panic pattern from
   20260713-175352.
3. **20260712-211352** (p80, refactor) Examples rework for testability.
   Baseline: 13 examples, 7 in the CI smoke list; 11/12 have the best
   assertions but are not in it; 4 have no harness. Audit
   keep/merge/remove/harness per example, harness the keepers with behavior
   assertions, document the bug-pin convention, pin this release's two bug
   fixes. Runs after the bug fixes so it can pin them.
4. **20260710-143138** (p20, ci, bug) Diagnose the GitHub-runner-only taffy
   panic in `03_scenario` and make `examples_smoke` blocking again - over
   the reworked example set, hence after 211352. CI experiments need
   pushes: prepare commits, then stop and ask the user before pushing.
5. **20260712-201603** (p15, chore, bcs) Root-fix bcs
   `InspectorDebugPlugin::on_add_camera` (first-camera-wins steals the
   inspector into RTT textures) upstream in ~/personal/bevy-common-systems,
   then bump nova's pinned rev (currently a35b74c = bcs master today, no
   drift) and delete nova's `keep_inspector_on_window_camera` workaround.
   User approved implementing the bcs change; pushing bcs to GitHub (needed
   before the rev bump builds) is a stop-and-ask.
6. **20260713-192510** (p5, chore) Release chore: version 0.5.1 -> 0.5.2,
   roll the changelog, one `chore(release)` commit, mirroring v0.5.1's
   ef1d7ad. No push without the user.

## Process

Each task: sprout worktree from master, /work with fail-first tests,
/review until APPROVE, squash-merge to master from the main checkout,
retro. Every task adds its own CHANGELOG.md Unreleased entry; the release
chore rolls them. Nothing is pushed anywhere without the user - the two
tasks that structurally need pushes (CI experiments, bcs rev bump) stop and
ask at that point.
